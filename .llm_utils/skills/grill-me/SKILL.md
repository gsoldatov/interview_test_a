---
name: grill-me
description: Interview user and explore the codebase to make or complete a plan for making updates to the codebase. Use when user wants to add a feature, writes a plan for making changes to the project or mentions "grill me".
---

Interview user relentlessly about every aspect of the plan which will solve the problem. Figure out the options for implementing the feature and move down through the design tree, resolving dependencies between decisions one-by-one, starting with the ones that might have the biggest impact on the solution.

Explore the codebase to understand its current state, verify user's assertions about it and answer potential questions about the solution.

Continue, until you've reached a shared understanding with user on WHAT and HOW must be changed in the project. Do not start changing the project, until you summarize the plan for adding the feature and user confirms it.
